 “правило трёх”

Когда в классе есть динамические ресурсы (например, new[], файлы, указатели и т. д.), компилятор не может догадаться, как правильно их копировать и чистить. Поэтому если ты сам определяешь один из трёх методов ниже — обязан определить все три:
 1. Деструктор
Освобождает память или другие ресурсы.
~~~C++
~MyString() {
    delete[] str;
}
~~~

 2. Конструктор копирования
Создаёт новый объект как копию существующего. Нужен, чтобы не копировать просто “указатель”, а реально дублировать данные.
~~~C++
MyString(const MyString& s) : size(s.size) {
    str = new char[size];
    memcpy(str, s.str, size);
}
~~~

 3. Оператор присваивания (operator=)
Вызывается, когда ты делаешь s1 = s2;. Тут надо сначала почистить старые данные s1, потом скопировать из s2.
~~~C++
MyString& operator=(const MyString& s) {
    if (this == &s) return *this; // защита от самоприсваивания
    
    delete[] str;
    size = s.size;
    str = new char[size];
    std::copy(s.str, s.str + size, str);
    return *this;
}
~~~


 Конструктор с параметрами

Ты можешь создавать строку определённого размера, заполненную каким-то символом:
~~~C++
MyString(size_t size, char c = '\0') {
    this->size = size;
    str = new char[size];
    memset(str, c, size);
}
~~~

memset просто заливает память одним байтом (c). Если c = '\0', то строка будет пустая.


 swap — друг на все времена

В C++ часто используют “идиому copy-swap” для оператора присваивания, чтобы упростить логику и защититься от ошибок:
~~~C++
void swap(MyString& s) {
    std::swap(size, s.size);
    std::swap(str, s.str);
}

MyString& operator=(MyString s) { // копия создаётся по значению
    swap(s);
    return *this;
}
~~~

Тут всё элегантно: копия временно создаётся (через конструктор копирования), а потом просто меняются местами данные. После выхода из функции временный объект сам удалится, а старые данные будут уничтожены вместе с ним.


Пример использования
~~~C++
int main() {
    MyString s1(5, 'a'); // "aaaaa"
    MyString s2 = s1;    // копия через конструктор копирования
    MyString s3(3, 'b'); // "bbb"
    s3 = s1;             // присваивание, вызывается operator=
}
~~~




# Списки_member_initializer_list

## Что это вообще такое

Когда ты создаёшь объект класса, у него могут быть:

- поля (переменные-члены),
    
- базовые классы (если наследуешься).
    

И всё это нужно инициализировать.  
Можно сделать это внутри конструктора, но C++ предлагает более элегантный (и эффективный) способ — список инициализации членов (member initializer list).

---

## Синтаксис

class MyClass {     int a;     double b;     std::string s;  public:     MyClass(int x, double y, std::string str)         : a(x), b(y), s(str) // ← вот это и есть список инициализации     {         // тело конструктора     } };

Вот эта часть : a(x), b(y), s(str) и есть member initializer list.

---

## Почему нельзя просто присвоить в теле конструктора

Можно, но тогда ты не инициализируешь, а сначала создаёшь, потом присваиваешь.  
Разница есть:

MyClass(int x, double y, std::string str) {     a = x; // тут a уже создана по умолчанию и только потом получает значение     b = y;     s = str; }

Для базовых типов вроде int это не страшно, но если у тебя, например, const поле или ссылка (&), ты вообще не сможешь присвоить — их можно только инициализировать при создании.

Пример:

```
class Test {     const int a;     int& ref; public:     Test(int x, int& r) : a(x), ref(r) {} // работает     // Test(int x, int& r) { a = x; ref = r; } // ошибка };
```



## Когда вызывается список инициализации

Он выполняется до входа в тело конструктора.  
И порядок инициализации зависит не от порядка в списке, а от порядка объявления полей в классе.

class Demo {     int x;     int y; public:     Demo() : y(10), x(y) {} // x инициализируется раньше y, потому что объявлен раньше };

Здесь x(y) — опасно, потому что y ещё не инициализирован.

---

## Как это работает с наследованием

Если класс наследуется, то сначала вызывается конструктор базового класса, и его тоже можно указать в списке:

class Base { public:     Base(int n) { std::cout << "Base " << n << "\n"; } };  class Derived : public Base {     int a; public:     Derived(int n) : Base(n), a(n * 2) {         std::cout << "Derived\n";     } };

---

## Плюсах

- Позволяет инициализировать const и ссылки.
- Эффективнее: без двойного присвоения.
- Даёт контроль над порядком инициализации
- Удобно при наследовании.

---

# Ошибки в C++ и процесс сборки программ

## Типы ошибок в программировании

### 1. Ошибки компиляции (Compilation Errors, CE)

**Определение:** Ошибки, которые компилятор обнаруживает на этапе перевода исходного кода в машинный. Программа не может быть запущена.

**Причины:** Несоответствие синтаксису и правилам языка C++

**Примеры:**
```cpp
int main() {
    cout << "Hello";  // Ошибка: не подключен <iostream> или отсутствует std::
}

int x = "строка";     // Ошибка типов: несовместимость типов данных
```

**Категории ошибок компиляции:**
- **Синтаксические ошибки** - неверное написание конструкций языка
- **Ошибки типов** - несовместимость типов данных
- **Ошибки имён** - использование необъявленных переменных или функций
- **Ошибки линковки** - объявлена функция, но отсутствует реализация

**Следствие:** Компилятор останавливает сборку, программа не создаётся

---

### 2. Ошибки времени выполнения (Runtime Errors, RE)

**Определение:** Ошибки, возникающие во время работы программы после успешной компиляции

**Причины:** Некорректные данные, неверные вычисления или действия, нарушающие правила выполнения

**Примеры:**
```cpp
int a = 0;
cout << 10 / a;        // Деление на ноль - RE

int arr[3] = {1, 2, 3};
cout << arr[10];       // Выход за границы массива - RE
```

**Типичные случаи:**
- Деление на ноль
- Доступ к неинициализированным или освобождённым указателям
- Выход за границы массива
- Переполнение памяти

**Следствие:** Программа компилируется, но при выполнении завершается с ошибкой (крашится)

---

### 3. Неопределённое поведение (Undefined Behavior, UB)

**Определение:** Ситуация, при которой стандарт языка C++ не определяет результат работы программы

**Причины:** Действия программиста нарушают правила стандарта языка

**Примеры:**
```cpp
int arr[3] = {1, 2, 3};
cout << arr[10];        // UB - выход за границы массива

int* p = nullptr;
cout << *p;            // UB - разыменование нулевого указателя

int x = INT_MAX;
x++;                   // UB - переполнение знакового целого
```

**Особенности:**
- Поведение непредсказуемо: от "работает нормально" до краша системы
- Программа может работать на одном компьютере и падать на другом
- Сложность отладки из-за непредсказуемости

**Следствие:** Поведение программы не поддаётся анализу

---

## Процесс сборки программ

### 1. Препроцессинг (Preprocessing)

**Суть:** Обработка директив, начинающихся с `#`

**Что происходит:**
- Подключаются заголовочные файлы (`#include`)
- Обрабатываются макросы (`#define`)
- Выполняются условия компиляции (`#ifdef`, `#ifndef`)

**Пример:**
```cpp
#include <iostream>
#define PI 3.14

int main() {
    std::cout << PI;
}
```

**Результат:** Создаётся расширенный исходный код с вставленным содержимым заголовочных файлов

---

### 2. Компиляция (Compilation)

**Суть:** Перевод расширенного исходного кода в объектный код

**Что происходит:**
- Проверка синтаксиса и семантики
- Преобразование кода в машинные инструкции
- Генерация объектных файлов

**Пример:** `main.cpp` → `main.o`

**Результат:** Объектные файлы (`.o` или `.obj`)

---

### 3. Линковка (Linking)

**Суть:** Сборка всех объектных файлов и библиотек в единый исполняемый файл

**Что происходит:**
- Склеиваются все `.o` файлы
- Подключаются статические библиотеки
- Разрешаются внешние ссылки (вызовы функций из других модулей)
- Проверяется наличие всех реализаций объявленных функций

**Пример:**
```cpp
// main.cpp
#include <iostream>
void hello();

int main() {
    hello();
}

// hello.cpp
#include <iostream>
void hello() {
    std::cout << "Hi!\n";
}
```

**Результат:** Исполняемый файл (`.exe` на Windows, без расширения на Linux)

---

### 4. Загрузка и исполнение (Loading & Execution)

**Суть:** Запуск программы операционной системой

**Что происходит:**
- Загрузка исполняемого файла в память
- Загрузка динамических библиотек (если используются)
- Подготовка стека, кучи и точек входа
- Передача управления функции `main`

**Результат:** Программа начинает выполнение

---

## Схема процесса сборки

```
Исходный код (.cpp, .h)
        │
        ▼
 [Препроцессор]
        │
        ▼
 Расширенный исходник
        │
        ▼
 [Компилятор]
        │
        ▼
 Объектный код (.o / .obj)
        │
        ▼
 [Линковщик]
        │
        ▼
 Исполняемый файл (.exe / бинарь)
        │
        ▼
 [Загрузка ОС]
        │
        ▼
 Выполнение программы
```

## Дополнительная информация

**Много файловые проекты:**
- Каждый `.cpp` компилируется отдельно
- Линковщик склеивает все объектные файлы

**Системы сборки:**
- **Make** и **CMake** автоматизируют процесс сборки
- Позволяют описывать правила сборки вместо ручной компиляции

**Динамические библиотеки:**
- Если используются `.dll` или `.so`, линковка может быть частично отложена до запуска программы

## Ключевые выводы

1. **Препроцессинг** - чистка и расширение кода
2. **Компиляция** - преобразование в объектные файлы  
3. **Линковка** - сборка в исполняемый файл
4. **Загрузка** - запуск программой ОС и выполнение

## Статические библиотеки

### Что такое статические библиотеки
- Набор скомпилированных объектных файлов (.o/.obj), объединенных в один файл
- Расширения:
  - `.lib` - Windows
  - `.a` - Linux
- Код "вшивается" в исполняемый файл на этапе линковки

### Преимущества и недостатки
**Плюсы:**
- Повторное использование кода
- Ускоренный запуск программы
- Нет внешних зависимостей

**Минусы:**
- Увеличенный размер исполняемого файла

### Процесс создания и использования

```cpp
// mathutils.cpp
int add(int a, int b) { return a + b; }

```

```bash
g++ -c mathutils.cpp -o mathutils.o
ar rcs libmath.a mathutils.o
```

Использование:
```cpp
// main.cpp
#include <iostream>
int add(int, int);

int main() {
    std::cout << add(2, 3);
}
```

Сборка:
```bash
g++ main.cpp -L. -lmath -o main
```

### Что такое динамические библиотеки
- Код подключается во время выполнения программы

  - `.dll` - Windows
  - `.so` - Linux
  - `.dylib` - macOS

### Преимущества и недостатки
**Плюсы:**
- Меньший размер исполняемого файла
- Обновления без пересборки программы
- Разделение кода между процессами

**Минусы:**
- Зависимость от внешних файлов
- Незначительное снижение производительности

## Класс std::vector

### Что такое vector
`std::vector` - динамический массив из стандартной библиотеки C++, который автоматически увеличивает размер при добавлении элементов.

### Основные методы

| Метод | Описание |
|-------|----------|
| `push_back(x)` | Добавляет элемент в конец |
| `pop_back()` | Удаляет последний элемент |
| `size()` | Возвращает текущий размер |
| `empty()` | Проверяет, пустой ли вектор |
| `clear()` | Очищает весь вектор |
| `resize(n)` | Изменяет размер вектора |
| `front() / back()` | Первый и последний элемент |
| `begin() / end()` | Итераторы для циклов |

```cpp
#include <iostream>

using namespace std;

int main() {
    vector<int> nums;
    
    nums.push_back(10);
    nums.push_back(20);
    nums.push_back(30);
    
    for (int i = 0; i < nums.size(); i++) {
        cout << nums[i] << " ";
    }
    // Вывод: 10 20 30
}
```

```cpp
vector<int> a = {1, 2, 3, 4};        // Инициализация списком

vector<int> c(10);                   // 10 элементов с значениями по умолчанию
```

```cpp
// С использованием итераторов
for (auto it = v.begin(); it != v.end(); ++it)
    cout << *it << " ";

// Range-based for loop
for (int x : v)
    cout << x << " ";
```

**Двумерный вектор (матрица):**
```cpp
vector<vector<int>> matrix(3, vector<int>(4, 0)); // Матрица 3x4 из нулей
```

### 1. Классы: базовая идея

Класс — это как шаблон для создания объектов.  
Типа ты создаёшь свою собственную “структуру данных” с переменными (поля) и функциями (методы).  
Всё, что описывает сущность — пихай туда.

class Student { public:     std::string name;     int age;      void sayHi() {         std::cout << "Привет, я " << name << "!" << std::endl;     } };

Теперь можно делать экземпляры этого класса:

Student s; s.name = "Дима"; s.age = 19; s.sayHi();

---

### 2. Структуры (`struct`)

В C++ struct и class почти одно и то же.  
Разница — по умолчанию:

- в struct всё public,
    
- в class всё private.
    

struct Point {     int x, y; };

class Point { private:     int x, y; };

Вот и вся магия.  
Раньше структуры были просто для хранения данных (как в C), а классы — для логики, но теперь это скорее вопрос стиля.

---

### 3. Инкапсуляция

Это умное слово означает: _не суй нос куда не надо_.  
То есть, скрывай внутренности класса от внешнего мира.  
Всё, что пользователю класса не нужно знать — делаешь private или protected.

class BankAccount { private:     double balance;  // скрыто  public:     BankAccount(double initial) {         balance = initial;     }      void deposit(double amount) {         balance += amount;     }      void withdraw(double amount) {         if (amount <= balance)             balance -= amount;     }      double getBalance() const {         return balance;     } };

Пользователь может вызвать методы, но не может напрямую лезть в balance и сломать всё к чертям.

---

### 4. Зачем вообще эта инкапсуляция

- Защищает данные от случайных (и тупых) изменений.
    
- Помогает управлять логикой — ты контролируешь, _как именно_ объект ведёт себя.
    
- Делает код проще для других — не нужно помнить, где там лежит каждая переменная.
    

---

### 5. Доступ: public / private / protected

| Модификатор | Где видно                   | Пример                |
| ----------- | --------------------------- | --------------------- |
